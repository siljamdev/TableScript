Libraries

To create a custom library with external functions, start by creating a static class with static methods:

using TabScript;

public static class CustomLib{	
	public static int sum1(int a){
		return a + 1;
	}
	
	public static string getDayOfTheWeek(int index){
		return index switch{
			1 => "Monday",
			2 => "Tuesday",
			3 => "Wednesday",
			4 => "Thursday",
			5 => "Friday",
			6 => "Saturday",
			7 => "Sunday",
			_ => null
		};
	}
}

Because TableScript functions always return Table and take Table as arguments, they need to be converted.
For the methods, the valid return types are:

  • void: Returns an empty Table
  • string: Returns a single element table. If the string is null, an empty table
  • int: Returns a n element table (new Table(int))
  • bool: Returns a 0 or 1 element table (Table.GetBool(bool))
  • Table: No conversion is done

For the arguments, they must be one of the following:

  • string: The table argument is reduced (Table.AsString())
  • int: The table argumet Length is used (Table.Length)
  • bool: The table truthiness(length > 0) is used (Table.Truthy)
  • Table: No conversion is done

Now, add a property to convert to a ResolvedImport. Once this conversion is done, there is no point in doing it again, so we store it in a private field.
Also, creating a list with functions and their description will make it easier.

public static (Delegate func, string description)[] AllFunctions => new (Delegate, string)[]{
	(sum1, "Sum 1 to a number"),
	(getDayOfTheWeek, "Get day of the week name by index")
};

static ResolvedImport compiled = null;

public static ResolvedImport AsImport {get{
	if(compiled == null){
		compiled = Library.BuildLibrary("customlib", AllFunctions);
	}
	return compiled;
}}

As we saw previously, we can use a custom IImportResolver.
In this example, we will implement a Resolver on top of the Standard Resolver that can resolve this new library.
This is the custom IImportResolver:

using TabScript;

class CustomImportResolver : StandardImportResolver{
	public override ResolvedImport Resolve(string import, string callingFilename){
		if(import == "customlib"){
			return CustomLib.AsImport;
		}
		return base.Resolve(import, callingFilename); //Safely handle anything that wasnt recognized
	}
}

Then, we just need the Main method:

using TabScript;

class Program{
	static void Main(){
		string path = "program.tbs";
		
		try{
			TableScript script = TableScript.FromSource(path, File.ReadAllText(path), new CustomImportResolver());
			
			script.Run();
		}catch(TabScriptException ex){
			Console.Error.WriteLine(ex);
		}
	}
}

Now, using this CustomImportResolver, we will have acess to sum1 and getDayOfTheWeek:

// IMPORTANT
// In this file, next to print() calls, the supposed output will be shown in a comment along with a commentary if needed

import stdlib;
import stdnum;
import customlib; //Our custom library!

print(sum1(3).toNum()); //4
print(getDayOfTheWeek(5)); //Friday

If there is any doubt about the signature of functions implemented in libraries, you can always print them out:

Console.WriteLine(CustomLib.AsImport);

This prints:

customlib
export function sum1(a){ EXTERN; } //Sum 1 to a number
export function getDayOfTheWeek(index){ EXTERN; } //Get day of the week name by index
