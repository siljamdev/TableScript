Multiple Files

Combining multiple files helps you to organize your code. How can we make it possible?
As we saw previously, we can use a custom IImportResolver.
In this example, we will implement a Resolver on top of the Standard Resolver that can resolve files and merge them together.
This is the custom IImportResolver:

using TabScript;

class CustomImportResolver : StandardImportResolver{
	string path; //Keep the path
	
	public CustomImportResolver(string p){
		path = p;
	}
	
	public override ResolvedImport Resolve(string import, string callingFilename){
		if(File.Exists(path + "/" + import)){
			return TableScript.SourceAsImport(path + "/" + import, File.ReadAllText(path + "/" + import), OnReport);
		}
		return base.Resolve(import, callingFilename); //Safely handle anything that wasnt recognized
	}
}

Then, we just need the Main method:

using TabScript;

class Program{
	static void Main(){
		string path = "program.tbs"; //Local path
		
		try{
			TableScript script = TableScript.FromSource(path, File.ReadAllText(path), new CustomImportResolver("."));
			
			script.Run();
		}catch(TabScriptException ex){
			Console.Error.WriteLine(ex);
		}
	}
}

When doing this, the functions marked export of the second file will be available for use.
Here is an example of a two-file program, intentionally complicated to show what can and cannot be done.

// IMPORTANT
// In this file, next to print() calls, the supposed output will be shown in a comment along with a commentary if needed

import stdlib; //Nothing happens for duplicated imports
import "multifiles2.tbs"; //Import the other file. Cyclic imports is not something to worry about

tab a = "Hello world"; //Global variable

tab b = "Greetings!"; //Global variable

print(greet()); //Greetings!
print(greet2()); //I have always been ready to greet you
print(greet3()); //I have always been ready to greet you
print(greet4()); //I have always been ready to greet you
print(greet5()); //I have always been ready to greet you
print(greet6()); //Greetings
print(bye());

function fun(){
    return b; //Accessing local variables
}

export function greet(){
    return fun(); //Can access function that isnt export
}

import stdlib; //Nothing happens for duplicated imports
import "multifiles1.tbs"; //Import the other file. Cyclic imports is not something to worry about
import stdnum; //Only in this file

tab a = "I have always been"; //Global variable

tab b = " ready to greet you"; //Global variable

print("Welcome!"); //This wont execute. Of the main body of imported scripts, only variable declarations are kept

//Returns "I have always been", "ready to greet you"
function fun(){
    return a + b; //Accessing local variables
}

//same name as function in other file
export function greet(){
    return fun(); //Can access function that isnt export
}

export function greet2(){
    return local::fun(); //Accesses local function
}

export function greet3(){
    return greet(); //Accesses the local function, locals are prioritised if unspecified import
}

export function greet4(){
    return local::greet(); //Accesses local function
}

export function greet5(){
    return multifiles2_tbs::greet(); //Accesses local function with its import name
}

export function greet6(){
    return multifiles1_tbs::greet(); //Accesses the function in the other file specifying the import
}

export function bye(){
    //Stdnum is only available in this file
    if stdnum::greater(stdnum::rand("0", "1"), "0.5"){
        print("Sad to see you go!");
    }else{
        print("Bye!");
    }
}
